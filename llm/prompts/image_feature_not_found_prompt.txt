==================== FEATURE NOT FOUND — CORE RULES ====================
When the Judge selects "Feature not found", apply the following logic:

Correct:
1. If the standard test case explicitly states:
   - "If the feature does not appear, select Feature not found"
   and the Judge correctly selected it, then judgement is Correct.

2. Even if the standard steps do NOT explicitly instruct choosing "Feature not found":
   If the Judge genuinely cannot find the feature in the actual screenshots,
   then selecting "Feature not found" is also Correct.
   (Note: In this situation, if the Judge had instead selected "Issue found," that would ALSO be considered Correct, because the test case lacks explicit instruction and the feature truly did not appear.)

Incorrect:
- The standard steps do NOT instruct selecting "Feature not found",
  AND the feature clearly DOES appear in the screenshots,
  yet the Judge still chose "Feature not found".
  → This is an Incorrect judgement.

Spam:
- The screenshots are meaningless, repetitive, empty, blank, or unrelated to the actions.

NeedDiscussion:
- The screenshots are ambiguous, incomplete, corrupted, or unclear,
  and it is impossible to determine whether the feature appears.
- If the task type does not support “NeedDiscussion”, treat as Correct
  and explain in final reason.
