==================== FEATURE NOT FOUND — CORE RULES ====================
EXPLICIT INSTRUCTION PRIORITY (READ FIRST, APPLY FIRST)
- If the standard explicitly instructs: "If the [feature/ad/top banner] does not appear, select 'Feature not found'", you MUST treat the Judge's selection of "Feature not found" as Correct when the screenshots do not show the feature.
- This explicit instruction overrides other rules. Do not mark Incorrect due to secondary mismatches when the absence condition is met.
- Case-level handling: when explicit instruction applies and the feature is absent, set final_result = "Correct" with a reason referencing the explicit instruction.
A Judge's decision of "Feature not found" must be evaluated based on the following rules:

------------------------------
✔ Correct (Case-Level Correct)
------------------------------
The Judge’s selection of “Feature not found” is considered Correct when ANY of the following are true:

1. **Explicit instruction in the test case** (PRIORITY)
  The standard test case explicitly states:
  - “If the feature/top banner/ad does not appear, select Feature not found”
  AND the screenshots confirm the feature is absent.
  → Correct (This rule overrides ordinary mismatch rules.)

2. **Feature truly does not appear (even if test case DOES NOT mention Feature not found)**
   - The feature genuinely does NOT appear in the screenshots.
   - The Judge could not find the feature.
   → Selecting “Feature not found” is Correct.
   NOTE:
   In such cases, if the Judge had selected "Issue found" instead,
   that would ALSO be considered Correct, because the test case gives no explicit rule.

3. **Feature is inherently randomized**
   Examples:
   - Search result modules (e.g., QnA block may or may not appear)
   - Dynamic feed sections
   If the screenshots confirm it did not appear, Judge’s decision is Correct.

--------------------------------
✖ Incorrect (Case-Level Incorrect)
--------------------------------
The Judge’s selection is Incorrect when ALL of the following are true:

1. The standard test case does NOT instruct selecting “Feature not found”.
2. The feature clearly DOES appear in the screenshots.
3. The Judge still selected “Feature not found”.

→ This is a misjudgment and must be marked Incorrect.

------------------------------
⚠ Spam
------------------------------
Mark as Spam when:
- Screenshots are meaningless, blank, repetitive, irrelevant,
- Or clearly unrelated to the expected operations.

------------------------------
⚠ NeedDiscussion
------------------------------
Choose NeedDiscussion if:
- Screenshots are incomplete, corrupted, too ambiguous,
- Or it is genuinely impossible to determine whether the feature appears.

If the task type does not support “NeedDiscussion”:
→ Treat as Correct and explain why in the final reason.
